RAW Data Requirements

Data classified as RAW DATA MUST meet ALL of the following requirements:
  1. Direct download: Directly downloadable file from official agency archive
  2. Official agency: Source must be from approved whitelist (NASA, ESA, JAXA, MPC)
  3. Not API-only: Must be archival file, not API response or computed values
  4. Provenance: Source URL, license/policy reference documented
  5. Checksum: SHA-256 checksum computed and verified
  6. Delay policy: Data release timing explicitly stated

Download Infrastructure

The RAW data download system includes:

  • Direct file download with content verification
  • SHA-256 checksum computation
  • Sidecar metadata (.meta.json) with provenance
  • Polite rate limiting (2+ seconds between requests)
  • Retry logic for transient failures
  • Storage structure: data/raw/<agency_id>/<dataset_id>/

See src/raw_download/ for implementation details.

Communication Policy: TRIZEL-AI does not provide direct messaging. All communication is conducted through verifiable channels: GitHub (issues, pull requests, discussions), ORCID (research profiles), and DOI-linked archival records (Zenodo, OSF, HAL).
