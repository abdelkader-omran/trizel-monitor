Communication Policy

IMPORTANT: TRIZEL-AI does not provide direct messaging.

All communication is conducted through verifiable channels ONLY:

  • GitHub: Issues, pull requests, and discussions at github.com/abdelkader-omran/trizel-monitor
  • ORCID: Research profiles with verified identities
  • DOI-linked archival records: Zenodo, OSF, HAL

No other communication channels are authorized.

RAW Data Policy

Classification Requirements

Data MAY ONLY be classified as RAW DATA if ALL conditions are met:

  1. Source is official agency from approved whitelist (NASA, ESA, JAXA, MPC)
  2. Direct download from archival system (not API summary)
  3. Verifiable provenance (source URL, checksum, policy reference)
  4. SHA-256 checksum computed and stored
  5. License or policy reference included
  6. Data release delay policy explicitly stated

Prohibited Classifications

  • API responses MUST NOT be classified as RAW_DATA
  • Computed values MUST NOT be classified as RAW_DATA
  • Real-time queries MUST be classified as SNAPSHOT

Verification Requirement

Before labeling any dataset "RAW_DATA", the system MUST provide verifiable proof:

  • Exact official endpoint URL
  • Direct file download possible without private credentials
  • File stored under data/raw/ with SHA-256
  • Sidecar .meta.json with provenance + license/policy + delay policy

Data Governance Framework

Version: 3.0.0

Status: AUTHORITATIVE

Core Principles

  • No interpretation: System does not modify, reinterpret, or alter official datasets
  • Evidence-first: Pointers + verifiable provenance only
  • Zero semantic drift: Strict definitions enforced
  • Single metadata block: Exactly ONE trizel_metadata block at root

Checksum Policy

  • SHA-256: REQUIRED for all data files
  • MD5: ABSOLUTELY FORBIDDEN (cryptographically broken)
  • CI MUST fail if MD5 is detected anywhere

Agency Status Policy

Active

Publicly downloadable RAW datasets exist and are implemented in download system. No credentials required.

Monitoring

Agency exists but raw download not implemented OR not publicly accessible without credentials. System monitors official channels for updates.

Limited

Only catalogs/metadata or derived products available without authentication.

Note: CNSA and ROSCOSMOS are marked "Monitoring" because publicly accessible RAW data downloads are not available without institutional credentials. This is an accurate reflection of their data access policies.

CI/CD Enforcement

All governance rules are enforced through automated CI validation:

  • Single metadata block validation
  • RAW_DATA classification requirements
  • Agency whitelist enforcement
  • MD5 prohibition check
  • Visual attributes correctness
  • Required fields validation

CI MUST fail if:

  • RAW_DATA claimed without meeting all requirements
  • MD5 checksum found anywhere
  • Multiple metadata blocks present
  • Unsupported agency used
  • Missing visual attributes

Backward Compatibility

Versions < 3.0.0 may contain snapshot data without explicit RAW_DATA guarantees.

For scientific reproducibility and verifiable raw data provenance, use v3.0.0 or later.

Migration from v2.x to v3.0.0:

  • Existing data files remain valid (backward compatible)
  • New RAW_DATA requirements apply to new data only
  • Agency registry now authoritative source of truth
  • UI provides verification interface for researchers
